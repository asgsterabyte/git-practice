1. 깃허브(서버 = remote repository)에서 복제(clone) -> 해당 프로젝트(= 폴더)를 가져오고 싶다( pull 아님)
-> git clone "링크"

2. 원하는 파일 작업(파일 생성, 파일 수정, 파일 삭제)함

3. 작업한 내용을 반영하기 위해서 staging area로 이동하기 위함
-> git add 파일 이름   (+ 모든 파일을 적용할 때는 *을 쓰기)
-> 파일 추적이 가능한 상태가 되고, 커밋할 준비가 됨

4. 실제 작업한 내용을 반영하기 위해 commit
-> git commit -m "원하는 메세지"  (원하는 메세지와 함께 커밋)
-> 로컬 환경에서 커밋(반영이 확정되었다)

5. 이걸 실제로 깃허브(서버)에 반영하기 위해서는 push
-> git push origin main (깃허브에 작업한 내용 반영)

6. 깃허브(서버)의 버전이 더 높을 때, 로컬과 버전 맞추고 싶을 때
-> git pull origin main
